### 2. tsc build-config gap closure (Item 50)

A `build:check` script was added to `package.json` and wired into `bun run check` between `typecheck` and `test`:

```
"build:check": "tsc -p packages/shared/tsconfig.build.json && tsc -p packages/sdk/tsconfig.build.json && tsc -p packages/cli/tsconfig.build.json"
"check": "bun run format:check && bun run lint && bun run typecheck && bun run build:check && bun run test"
```

This closes the gap that caused the Sprint 14 CI failure — per-package `tsconfig.build.json` compilation is now part of the local verification loop. The Scrum Master BOARD item from the Sprint 14 retrospective was moved to Closed.

### 3. Reusable workflow object injection (Item 48)

`usesWorkflow()` now accepts `WorkflowBuilder` and `WorkflowDefinition` objects in addition to `WorkflowRef` strings via the `ReusableWorkflowSource` union type:

```typescript
const deploy = defineWorkflow({ id: createWorkflowId('deploy'), name: 'Deploy' })
  .onWorkflowCall({ inputs: { env: { type: 'string', required: true } } });

const ci = defineWorkflow({ id: createWorkflowId('ci'), name: 'CI' })
  .onPush()
  .addJob(createJobId('run_deploy'), job => {
    job.usesWorkflow(deploy, { with: { env: 'prod' }, secrets: 'inherit' });
  })
  .build();

// Result: uses: ./.github/workflows/deploy.yml, with: { env: "prod" }, secrets: inherit
```

Key behaviors verified:

- WorkflowBuilder → derives `./.github/workflows/${id}.yml`
- WorkflowDefinition (built) → derives `./.github/workflows/${id}.yml`
- Missing `workflow_call` trigger → validation error at caller's `build()` time
- Definition-order independent — `onWorkflowCall()` can be called after `usesWorkflow()`
- `with` and `secrets` forwarding works with object injection
- Existing `WorkflowRef` string usage unchanged (backward compatible)
- `ReusableWorkflowSource` type exported from SDK public API
