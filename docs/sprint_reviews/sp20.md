# Sprint Review: Sprint 20

## Summary

Sprint 20 delivered all committed backlog items (`Ad hoc A1`, Items 67, 68, 69, 70a, 70b, 71, and 72) for a total of 18/18 committed story points. Item PRs #78 through #85 were merged into `sprint-20`, and the sprint closeout PR #86 (`sprint-20` -> `main`) was merged on 2026-04-04.

The increment materially expanded the product surface in three areas: CLI ergonomics (`-i` / `-o` aliases and inferred default output path for `ghawb render`), syntax and product-surface coverage (niche trigger closure plus the initial composite-action authoring slice), and practical authoring ergonomics (10 additional typed action wrappers plus README / COOKBOOK examples).

No committed sprint item carried over. The active product backlog remains Item 73.

## Increment Demo

### 1. CLI inferred output path for repository-local workflow rendering

Command:

```sh
bun run packages/cli/src/bin.ts render -i workflows/ci.ts
```

Result:

- The command completed without an explicit `--output`.
- The CLI rendered the repository-local default destination `.github/workflows/ci.yml`.
- This confirms the Sprint 20 contract that output-path inference is supported for `ghawb render` on the documented `workflows/<name>.ts` path.

### 2. Typed action wrappers now emit pinned first-party action refs with serialized inputs

Command:

```sh
bun -e 'import { createJobId, createWorkflowId } from "./packages/shared/src/index.ts"; import { defineWorkflow, createWorkflowRenderPayload } from "./packages/sdk/src/index.ts"; import { actionsCache, actionsSetupPython } from "./packages/typed-actions/src/index.ts"; const workflow = defineWorkflow({ id: createWorkflowId("typed_demo"), name: "Typed Demo" }).onPush().addJob(createJobId("test"), (job) => { job.runsOn("ubuntu-latest").uses(actionsSetupPython({ pythonVersion: "3.12" }), "Setup Python").uses(actionsCache({ path: ["~/.cache/pip"], key: "pip-${{ runner.os }}" }), "Cache"); }).build(); console.log(JSON.stringify(createWorkflowRenderPayload(workflow).jobs.test.steps, null, 2));'
```

Result:

- `actionsSetupPython()` rendered `actions/setup-python@v5` with `python-version: "3.12"`.
- `actionsCache()` rendered `actions/cache@v4` with serialized `path` and `key` inputs.
- This demonstrates the Sprint 20 typed-wrapper expansion beyond the original minimal wrapper set.

### 3. Composite action authoring renders the first supported `action.yml` payload slice

Command:

```sh
bun -e 'import { defineCompositeAction, createCompositeActionRenderPayload } from "./packages/composite-actions/src/index.ts"; const action = defineCompositeAction({ name: "Echo", description: "Demo composite action" }).uses("actions/checkout@v4", "Checkout").run("echo ok", { name: "Echo", shell: "bash", workingDirectory: "./scripts" }).build(); console.log(JSON.stringify(createCompositeActionRenderPayload(action), null, 2));'
```

Result:

- The rendered payload included top-level `name`, `description`, and `runs`.
- `runs.using` resolved to `"composite"`.
- Ordered composite steps rendered both a `uses` step and a `run` step with `shell` plus `working-directory`.
- This confirms the Sprint 20 initial composite-action authoring slice selected by ADR 0004.

### 4. Full-sprint verification and post-closeout repair

Commands:

```sh
bun run coverage
bun run check
gh pr view 86 --json mergedAt,statusCheckRollup,url
gh pr view 87 --json state,statusCheckRollup,url
```

Result:

- `bun run coverage` and `bun run check` passed on the review branch after the post-closeout CI repair.
- PR #86 was merged at `2026-04-04T16:51:43Z`, while its hosted `check` run had already completed with `FAILURE` at `2026-04-04T16:51:42Z`.
- The failure was not a flaky transient; it exposed stale workflow build commands that no longer matched the current package build surface after Sprint 20.
- Follow-up PR #87 repaired the workflow build step and restored the coverage gate, and its hosted `check` run completed with `SUCCESS` before merge.

## Findings

### Confirmed finding: the sprint increment delivered its committed feature scope

The delivered APIs, CLI behavior, documentation, and tests align with the committed Sprint 20 backlog. No committed item is missing from the implemented repository state, and no backlog item requires carry-over based on feature-scope incompleteness.

### Confirmed finding: sprint closeout merged despite a failing hosted CI check

Sprint 20 did not meet the repository's operational quality bar at closeout. The sprint closeout PR #86 was merged after GitHub Actions had already reported a failed `check` run. The failure was caused by stale hard-coded package build commands in the committed workflow definition, which had not been updated when `@ghawb/composite-actions` was added during the sprint. This was a real defect in the repository delivery process, not a harmless red build.

### Confirmed finding: the hosted coverage gate also needed repair once CI was corrected

After the stale build-step issue was fixed, hosted coverage still aborted because the repository enforced 100% lines/statements/functions while a small set of SDK serializer/helper branches no longer had direct test hits. The fix in PR #87 restored the gate by adding explicit tests for the uncovered Sprint 20-era branches rather than weakening thresholds.

### Residual risk: closeout process was corrected, but the underlying process gap remains important

The immediate repository-operability risk has been removed because PR #87 is now merged and hosted CI is green again. The remaining concern is procedural: Sprint 20 proved that local green checks were incorrectly treated as sufficient for sprint closeout. That gap should remain visible in team process and future sprint-closeout discipline even though the concrete CI defect is fixed.

## Product Owner Decision — Aoi Sakamoto

- Sprint 20 delivered all committed scope (18/18 SP, 8/8 items). No feature-scope carry-over is required.
- The next active backlog priority remains Item 73. No active-backlog reordering is needed.
- Sprint 20 should be accepted both for delivered product scope and for the current hardened repository state now that PR #87 is merged and hosted CI is green again.
- Future sprint closeout PRs must not be merged on the strength of local checks alone. Hosted GitHub Actions status is part of the release and closeout gate for this repository.
- The product direction chosen during Sprint 20 is retained: composite actions remain outside `@ghawb/sdk` behind a separate opt-in package, and typed action wrappers remain an additive opt-in ergonomics layer rather than part of the core workflow AST surface.
